# 快速开始指南

本指南将帮助你快速上手 AWS Auto Registration 扩展。

## 📋 前置要求

- ✅ Chrome 浏览器（版本 88+）
- ✅ 一个 Gmail 邮箱（Gmail 别名模式）或临时邮箱服务（完全自动化模式）
- ✅ （可选）CAPTCHA 服务 API Key（如果需要自动解决验证码）

---

## 🚀 5 分钟快速开始（Gmail 别名模式）

### 步骤 1：安装扩展

1. 克隆或下载项目
   ```bash
   git clone https://github.com/Specia1z/AWS-BuildID-Auto-For-Ext.git
   ```

2. 打开 Chrome 扩展页面
   - 访问 `chrome://extensions/`
   - 开启右上角「开发者模式」

3. 加载扩展
   - 点击「加载已解压的扩展程序」
   - 选择项目根目录

4. **重要**：启用无痕模式
   - 在扩展卡片上，点击「详细信息」
   - 找到「在无痕模式下启用」，**必须开启**
   - 刷新扩展（点击刷新图标 🔄）

### 步骤 2：配置 Gmail 地址

1. 点击扩展图标打开弹窗
2. 在「邮箱配置」区域：
   - 确保选择「Gmail 别名」模式
   - 输入你的 Gmail 地址（例如：`example@gmail.com`）
   - 点击「保存」按钮
3. 看到 ✓ 已保存 提示即配置成功

### 步骤 3：开始注册

1. 设置参数：
   - 注册数量：1（建议先测试 1 个）
   - 并发窗口：1（Gmail 模式建议设为 1）

2. 点击「开始注册」按钮

3. 等待无痕窗口打开，扩展会自动填写表单

4. **手动操作**：
   - 打开你的 Gmail 收件箱
   - 找到 AWS 发送的验证码邮件
   - 在注册页面手动输入验证码

5. 等待注册完成，查看结果

### 步骤 4：查看结果

1. 在「注册历史」区域查看注册记录
2. 点击「验证」按钮验证 Token 状态
3. 点击「JSON」或「CSV」按钮导出账号信息

## 🎯 进阶使用（完全自动化）

### 前置准备

1. **搭建临时邮箱服务**
   - 推荐使用 [Cloudflare Email Worker](https://github.com/TBXark/cloudflare-email-worker)
   - 或购买临时邮箱 API 服务

2. **（可选）购买 CAPTCHA 服务**
   - 推荐 [YesCaptcha](https://yescaptcha.com/)（性价比高）
   - 或 [CapSolver](https://www.capsolver.com/)（速度快）
   - 或搭建本地 Solver（免费）

### 配置步骤

#### 1. 配置临时邮箱

1. 在扩展弹窗中，选择「临时邮箱」模式
2. 填写配置：
   ```
   Worker Domain: mailfly.codeforge.top
   Email Domain: example.com
   Admin Password: your-admin-password
   ```
3. 点击「保存配置」

#### 2. 配置 CAPTCHA（可选）

1. 启用「CAPTCHA 自动解决」开关
2. 选择服务商（例如：YesCaptcha）
3. 填写 API Key
4. 点击「保存配置」

#### 3. 开始批量注册

1. 设置参数：
   - 注册数量：10（可以设置更多）
   - 并发窗口：2-3（临时邮箱模式支持更高并发）

2. 点击「开始注册」

3. **完全自动化**，无需人工干预，等待完成即可

4. 查看注册结果和 Token 状态

## 📊 功能说明

### Token 验证

注册完成后，Token 默认状态为「未验证」，需要手动验证：

1. 点击「验证」按钮
2. 等待批量验证完成
3. 查看验证结果：
   - 🟢 有效：Token 正常可用
   - 🟡 封禁：账号被临时封禁
   - 🟠 过期：Token 已过期
   - 🔴 无效：账号无效或被删除

### 导出账号

- **JSON 导出**：仅导出有效和未验证的 Token（自动过滤封禁/过期/无效）
- **CSV 导出**：导出完整信息，包含 `token_status` 列

### Kiro IDE 同步

1. 在历史记录中找到成功注册的记录
2. 点击「Kiro」按钮
3. 插件会自动检测操作系统并生成命令
4. 在对应终端中粘贴并执行命令
5. 重启 Kiro IDE 即可使用

## ⚠️ 常见问题

### 提示"创建无痕窗口失败"

**解决方案**：
1. 访问 `chrome://extensions/`
2. 找到本扩展，点击「详细信息」
3. 开启「在无痕模式下启用」
4. 刷新扩展

### 收不到验证码邮件

**解决方案**：
1. 检查 Gmail 的「垃圾邮件」文件夹
2. 检查「所有邮件」确保邮件未被过滤
3. 确认 Gmail 地址输入正确

### 临时邮箱无法获取验证码

**解决方案**：
1. 检查临时邮箱服务配置是否正确
2. 测试邮箱服务是否正常运行
3. 查看浏览器控制台（F12）的错误信息
4. 尝试切换回 Gmail 别名模式

### CAPTCHA 解决失败

**解决方案**：
1. 检查 API Key 是否有效
2. 检查账户余额是否充足
3. 尝试切换其他服务商
4. 暂时禁用 CAPTCHA 自动解决

## 💡 使用技巧

### 提高成功率

1. **使用临时邮箱模式**：避免 Gmail 别名被识别
2. **启用 CAPTCHA 自动解决**：减少人工干预
3. **合理设置并发**：避免过高并发导致限流
4. **定期验证 Token**：及时发现失效账号

### 降低成本

1. **使用 Gmail 别名模式**：完全免费
2. **使用本地 CAPTCHA Solver**：无需购买 API
3. **选择性价比高的服务商**：如 CapSolver（$0.8/1000 次）

### 批量管理

1. **定期导出账号**：备份 Token 信息
2. **使用 CSV 格式**：方便在 Excel 中管理
3. **标记 Token 状态**：区分有效/无效账号
